# GitBook 本地启动指南

## 方式一：使用 Docker（推荐）

### 启动服务
```bash
cd /Users/ls/Documents/blog/blogs
docker-compose up
```

### 后台运行
```bash
docker-compose up -d
```

### 停止服务
```bash
docker-compose down
```

启动后访问：**http://localhost:4000**

---

## 方式二：本地安装 GitBook CLI

### 1. 安装 Node.js
确保已安装 Node.js（建议版本 10.x 或 11.x）

### 2. 安装 GitBook CLI
```bash
npm install -g gitbook-cli
```

### 3. 安装 GitBook 依赖
```bash
cd /Users/ls/Documents/blog/blogs
gitbook install
```

### 4. 启动服务
```bash
gitbook serve
```

启动后访问：**http://localhost:4000**

### 5. 构建静态文件（可选）
如果需要生成静态 HTML 文件：
```bash
gitbook build
```
构建后的文件会在 `_book` 目录中。

---

## 常见问题

### 端口被占用
如果 4000 端口被占用，可以修改端口：
- Docker 方式：修改 `docker-compose.yml` 中的端口映射，如 `"4001:4000"`
- 本地方式：使用 `gitbook serve --port 4001`

### 插件安装失败
如果 `gitbook install` 失败，可能需要：
1. 检查网络连接
2. 清除缓存：`gitbook clean`
3. 重新安装：`gitbook install`

### Docker 镜像构建失败
如果 Docker 构建时 npm 源访问有问题，可以：
1. 修改 Dockerfile 中的 npm 源配置
2. 或使用本地方式启动
